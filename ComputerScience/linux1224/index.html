# Ch5 Linux 的文件权限与目录配置

# 使用者与群组

Linux 一般将文件可存取的身份分为三个类别，分别是 owner/group/others，且三种身份各有
read/write/execute 等权限。而 root 可以访问所有文件。

# 文件权限

ls -al ：查看当前目录下文件（夹）的详细的权限与属性 （包含隐藏文件）

字段含义：

档案类型为 d 时表示文件夹， - 表示文件， l 表示为连结档（link file）， b 表示装置文件里面的可供储存的接口设备（可随机存取装置）， c 表示装置文件里面的串行端口设备，例如键盘、鼠标（一次性读取装置）。

对文件而言：

  • 可读 (r) 代表读取文件内容
  • 可执行 (x) 表示是否具有执行的能力
  • 可写 (w) 权限代表具有写入 / 编辑 / 新增 / 修改文件的内容的权限，但并不具备有删除该文件本身的权限

对目录而言：

  • 可读 (r) 代表读取目录结构列表，可以 ls 查询该目录下的文件名数据
  • 可执行 (x) 表示用户能否进入该目录成为工作目录，即 cd 进入
  • 可写 (w) 权限代表具有异动该目录结构列表的权限，即
    • 建立新的文件与目录
    • 删除已经存在的文件与目录 (不论该文件的权限为何！)
    • 将已存在的文件或目录进行更名
    • 搬移该目录内的文件、目录位置
      开放目录给任何人浏览时，应该至少也要给予 r 及 x 的权限，但 w 权限不可随便给！

案例：
假设两个档名，分别是 /dir1/file1 和 /dir2 。现在在系统使用 riolu 这个账号，那么这个账号针对 /dir1 , /dir1/file1 , /dir2 这三个档名来说，分别需要哪些最小的权限才能达成各项任务？

实际上没有 r 权限时，只要知道目录下的文件名就可以操作！只是，没有 r 的话，使用 tab 时，他就无法自动帮你补齐档名了！

# 修改权限

# 群组 chgrp

记得使用 root 权限修改哦。
要被改变的组名必须要在 /etc/group 文件内存在。
用法： chgrp [-R] grpname dir/filename 。grp 是 group 的缩写， -R 表示目录下所有文件递归（recursive）修改。

# 拥有者 chown

用法：
chown [-R] usrname dir/filename
chown [-R] usrname:grpname dir/filename
chown [-R] usrname.grpname dir/filename
chown [-R] .grpname dir/filename

# 权限 chmod

一种是用数字修改，但我觉得符号改比较人性化～

用法：

chmod u g o a(all) +(加入) -(除去) =(设定) r w x 文件或目录

如： chmod u=rwx,go+rx .bashrc

# 文件名规则

  1. 前面加 . 是隐藏文件
  2. 权限组开头为 l 的是链接档，类似于快捷方式
  3. 最好不要含 * ? > < ; & ! [ ] | \ ' " \( \) \{ \} - ` 等符号

# 目录配置

一般 Linux 下有这样的目录约定：

  1. / 根目录
  2. /usr(unix software resource) 软件放置处
  3. /var(variable) 与系统运作过程有关
  4. /opt 第三方软件，以前也在 /usr/local 下
  5. /etc 配置文件
  6. lost+found 并不是回收处，而是当文件系统发生错误时，将一些遗失的片段放置到这个目录下。

更多的可以去搜 Filesystem Hierarchy Standard (FHS)。
